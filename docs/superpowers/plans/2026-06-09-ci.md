# GitHub Actions CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 MessageFlight 添加最小 CI 工作流（Linux 容器跨 5 个 Python 版本做静态语法检查），push/PR 双向触发，并在两个 README 顶部加状态徽章。

**Architecture:** GitHub Actions 工作流 + Ubuntu runner + Python 矩阵。`winsdk` 装不上 Linux 容器，因此跳过依赖安装，只对脚本和 `pyproject.toml` 做静态校验。

**Tech Stack:** GitHub Actions, Python 3.8–3.12, py_compile, ast, tomli

## Task 1: 创建 CI 工作流文件

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: 创建目录**

```bash
New-Item -ItemType Directory -Path ".github\workflows" -Force | Out-Null
```

- [ ] **Step 2: 写入 ci.yml**

写入以下内容到 `.github/workflows/ci.yml`：

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  lint:
    name: Static checks (Python ${{ matrix.python-version }})
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        python-version: ["3.8", "3.9", "3.10", "3.11", "3.12"]

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
          cache: pip

      - name: Compile-check message_flight.py
        run: python -m py_compile message_flight.py

      - name: AST-parse message_flight.py
        run: python -c "import ast; ast.parse(open('message_flight.py').read())"

      - name: Validate pyproject.toml
        run: |
          pip install tomli
          python -c "import tomli; tomli.loads(open('pyproject.toml').read())"
```

- [ ] **Step 3: 验证文件存在并显示内容**

```bash
Get-Content .github/workflows/ci.yml
```

预期：YAML 内容完整，含 `name: CI`、matrix 5 个版本、3 个检查步骤。

- [ ] **Step 4: 提交**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add static-check workflow across Python 3.8-3.12"
```
