# GitHub Actions CI 设计

## 目标

为 MessageFlight 添加最小 CI：Linux 容器上对 `message_flight.py` 做静态语法检查，跨 5 个 Python 版本矩阵，push/PR 双向触发，并在两个 README 顶部加状态徽章。

## 背景

- 单脚本项目：`message_flight.py`（PyQt6 + winsdk）
- 仅 Windows 可运行：`winsdk` 是 Windows 专属
- 已有 `pyproject.toml`（requires-python `>=3.8`）
- CI 不能装 `winsdk`、不能启动 GUI；唯一可行的是静态检查

## 设计

### 工作流文件

- 路径：`.github/workflows/ci.yml`
- 工作流名：`CI`
- 触发：
  - `push` 到 `main`
  - `pull_request` 到 `main`

### Job

- 名称：`lint`
- 运行环境：`ubuntu-latest`
- 策略（matrix）：
  ```yaml
  python-version: ["3.8", "3.9", "3.10", "3.11", "3.12"]
  ```
- 失败即快速失败（无特殊 continue-on-error）

### 步骤

1. `actions/checkout@v4`
2. `actions/setup-python@v5`：
   - `python-version: ${{ matrix.python-version }}`
   - `cache: 'pip'`
3. 编译检查主脚本：
   ```bash
   python -m py_compile message_flight.py
   ```
4. 用 AST 解析脚本（独立于 py_compile，验证脚本可被解析为合法 Python）：
   ```bash
   python -c "import ast; ast.parse(open('message_flight.py').read())"
   ```
5. 验证 `pyproject.toml` 是合法 TOML（用 `tomli`，跨版本一致）：
   ```bash
   pip install tomli && python -c "import tomli; tomli.loads(open('pyproject.toml').read())"
   ```

### 不做的事

- 不安装项目依赖（`pip install .` 会因 `winsdk` 在 Linux 装不上而失败）
- 不启动 GUI
- 不跑测试（项目目前无测试代码）
